02 · Key findings
Other Relevant Info: near-total failure
80.7% error rate — the worst class by far. With F1 of only 0.285, the model hemorrhages 353 samples into "Not Humanitarian" alone. This class is a semantic catch-all, making it inherently ambiguous for zero-shot prompting.
Model is overconfident on wrong predictions
59% of all errors were made at confidence ≥ 0.90. At conf=0.70 only 12.5% accuracy — the model's confidence scores are poorly calibrated and do not track correctness reliably.
Requests/Needs is a rare-class failure
63.6% error rate on a class with only 55 samples. The model defaults to "Rescue/Donation" (28 errors) likely because surface keywords overlap. Class imbalance amplifies the failure.
Injured/Dead is the anchor class
F1 = 0.893 — the best-performing class with the largest support (1,362). Death-count and casualty language is unambiguous and likely well-represented in the LLM's pretraining.
Not Humanitarian bleeds into Sympathy
68 "Not Humanitarian" tweets were predicted as "Sympathy/Support." Emotional, opinionated tweets about the disaster are semantically close to sympathy — the LLM cannot distinguish social commentary from emotional support.
Missing People confused with Injured/Dead
32 / 125 missing-person tweets predicted as "Injured/Dead." Co-occurrence of names + disaster context drives the model toward the more frequent fatality class — a critical real-world error for emergency response.
